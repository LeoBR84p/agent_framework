Step 1 — Repository & Configuration

Fork the template and configure project identity files.

  • Fork or use template from GitHub
    Create a new repo from the AIz Intel template. One repo per project.
  • Fill in project_config.json
    Set name, tagline, stage, lastUpdated, team members, allowedUsers, and investorAllowedUsers.
  • Replace assets/logo.svg
    Upload your project logo. Recommended: SVG, 64×64px, transparent background.
  • Add assets/favicon.ico
    Generate a 32×32 ICO file from your logo and place it in assets/.
  • Customize assets/project_theme.css
    Override --brand-primary, --brand-accent, and font variables to match your brand.

Step 2 — Firebase Authentication

Set up Google OAuth and Firestore allowlist for access control.

  • Create Firebase project at console.firebase.google.com
    Enable Google Authentication under Build → Authentication → Sign-in method.
  • Fill window.FIREBASE_CONFIG in all HTML files
    Copy your Firebase web app config from Project Settings and replace all FILL placeholders. Files: auth.html, index.html, and all division pages.
  • Create Firestore allowedUsers collection
    For each authorized user, create a document with their email as the document ID. No fields required — existence check only.
  • Set Firestore security rules
    Allow authenticated users to read allowedUsers/{email} documents only. Deny all write access from client.
  • Update .firebaserc with your project ID
    Replace FILL_FIREBASE_PROJECT_ID with your actual Firebase project ID.

Step 3 — Vercel Deployment

Deploy as a static site on Vercel with auto-deploy on push.

  • Import GitHub repo into Vercel
    Go to vercel.com → New Project → Import GitHub repository. Select your forked repo.
  • Configure Vercel project settings
    Framework Preset: Other. Root directory: / (root). Output directory: . (dot). No build command needed.
  • Add custom domain (optional)
    In Vercel project → Settings → Domains. Add your domain and configure DNS records.
  • Add Vercel domain to Firebase authorized domains
    In Firebase console → Authentication → Settings → Authorized domains. Add your vercel.app URL and custom domain.

Step 4 — Business Division Content

Fill in all business-critical documents. Pricing must match Economics.

  • business/idea.html
    Fill: Problem Statement, Target Audience, Proposed Solution, UVP, Key Assumptions, Validation Status.
  • business/business_plan.html
    Fill: Executive Summary, BMC (9 blocks), TAM/SAM/SOM, Revenue Model, Team, Risk Matrix.
  • business/economics.html
    Fill: S-Curve Projections, Break-Even, CAC/LTV/Churn, Cash Flow Scenarios, Price Adequacy.
  • business/pricing.html
    Fill: Pricing Tiers, Competitor Price Matrix, Value Perception. Ensure alignment with economics.html.
  • business/competitor_analysis.html
    Fill: Competitor Cards, Feature Matrix, Positioning Map, Relative SWOT.
  • business/launch_plan.html
    Fill: Launch Phases, Early Adopter Strategy, Channels, Partnerships, Ad Campaigns.
  • business/marketing.html
    Fill: ICP Definition, Messaging Matrix, Channel Strategy, Campaign Calendar.
  • business/risk.html
    Fill: Risk Register, Heat Map, Category Taxonomy, Mitigation Status. Cross-reference business_plan.html.

Step 5 — Technology Division Content

  • technology/tech_plan.html
    Fill: Architecture Diagram, Stack Decision Log, Sprint Checklists, API Surface Summary.
  • technology/zero_hero.html
    Fill: Environment Setup, Bootstrap Commands, Architecture Decisions, Gotchas & Workarounds.
  • technology/db_schema.html
    Fill: Table Definitions, Relationships, Indexes, Migration History.
  • technology/roadmap.html
    Fill: Timeline Swimlanes, Milestone Definitions. Sprint status must align with okr_kpi.html.

Step 6 — Brand & Operations

  • brand/brand_guidelines.html
    Fill: Logo Usage, Color Palette Tokens, Typography System, Copy & Tone of Voice.
  • operations/okr_kpi.html
    Fill: Quarterly OKRs with Key Results, KPI Dashboard. Must align with roadmap.html sprint status.
  • Legal documents (ToS, Privacy, Cookie, DPA/SLA)
    Complete all four legal documents with counsel review. Ensure LGPD/GDPR compliance sections are filled.
  • Initialize metrics.json with real values
    Replace null values with actual metrics data. Update monthly or as data changes.

Step 7 — Investor & Fundraising

  • investor-relations/pitch.html
    Fill: Problem, Solution, Market Size, Business Model, Traction, Team, Financials, Use of Funds, Cap Table, The Ask.
  • investor/due_diligence.html
    Fill: DD Checklist across legal, financial, technical, team, market, product categories.
  • Test investor/index.html NDA gate
    Open investor/index.html in incognito. Verify NDA acceptance is required. Verify links work.

Step 8 — Final Checks & Go-Live

  • Test auth flow end-to-end
    Sign out, visit a protected page, verify redirect to auth.html, sign in, verify redirect back.
  • Test on mobile (375px width)
    Verify sidebar toggle works, nav collapses, all text is readable, no horizontal scroll.
  • Test print/PDF export on a key document
    Open business_plan.html, press Ctrl+P, verify sidebar is hidden and content is clean.
  • Test dark/light mode toggle
    Toggle theme from sidebar, verify preference persists after page reload.
  • Test global search
    Visit search.html, type a keyword, verify results appear with correct titles and links.
  • Push to main and verify Vercel auto-deploy
    Commit all changes, push to main branch, verify Vercel deploys successfully, visit production URL.
